去年我們進行了用戶訪談並發佈了[一份問卷調查](https://www.surveymonkey.co.uk/r/DDZWQDJ)，以深入了解人們如何使用React Native文檔。透過24場訪談和超過3000份問卷回饋的數據與指引，我們得以改進React Native的文檔，並興奮地在此分享這些進展：

- **[全新入門指南](https://reactnative.dev/docs/getting-started)** 我們推出了新的入門文檔，向沒有移動開發背景的人[解釋什麼是原生組件](https://reactnative.dev/docs/intro-react-native-components)。同時也包含了[React的複習/介紹](https://reactnative.dev/docs/intro-react)，幫助初次接觸React的開發者！
- **[全新測試指南](https://reactnative.dev/docs/testing-overview)** 我們與Vojtech Novak合作創建了帶插圖的測試指南，向應用開發者介紹不同類型的測試策略及其在React Native工作流中的應用。
- **[全新安全指南](https://reactnative.dev/docs/security)** 我們與Kadi Kraman合作創建了帶插圖的安全指南，解釋React Native中的安全基礎知識並提供最佳實踐方案。
- **更多插圖** 我們新增了精美的插圖，包括新的[`Pressable`](https://reactnative.dev/docs/pressable)和[React Native組件介紹](https://reactnative.dev/docs/intro-react-native-components)文檔。
- **https://reactnative.dev** 經過5年，我們終於遷移到專屬域名！**reactnative.dev**比之前的**github.io**地址更容易從瀏覽器欄自動補全，也更便於輸入！
- **網站設計與架構改進** 我們更新了設計和網站架構，以更好地展示和分類指南內容，並提升API參考文檔的可讀性。特別感謝[Bartosz Kaszubowski](https://github.com/Simek)的細緻工作和協作，使許多改進得以快速實現！
- **更新的核心組件與API文檔** 我們舉辦了[文檔更新活動](https://github.com/facebook/react-native-website/issues/1579)來更新參考文檔！感謝以下參與者及其他貢獻者，我們得以全面更新文檔並為所有內容添加Snack示例，趕上0.62版本發佈：[Marta Dabrowka](https://twitter.com/martadabrowka)、[Abraham Nnaji](https://twitter.com/nnajiabraham)、[Ahmed Talaat El-Hager](https://twitter.com/ahmdtalat)、[Mohamed Abdel Nasser Abdou](https://twitter.com/mohamedsgap)、[Danilo Britto](https://twitter.com/danilobrinu)、[Mitul Savani](https://twitter.com/mitulsavani)、[Kaio Duarte](https://twitter.com/kaiodduarte)、[Pablo Espinosa](https://twitter.com/espipj)、[Jesse Katsumata](https://twitter.com/natural_clar)、[I Gede Agastya Darma Laksana](https://twitter.com/gedeagas)、[Sebastião Bruno Kiafuka Fernando](https://twitter.com/bruno_kiafuka)、[Miguel Bolivar](https://twitter.com/Darking360)、[Dani Akash](https://twitter.com/dani_akash_)、[Luiz Celso de Faria Alves](https://twitter.com/_eucelso)和[Bartosz Kaszubowski](https://twitter.com/simek)。有了他們的貢獻，這些是目前最好且最新的React Native文檔！
- **持續提交PR！** 我們能持續將每週開放的PR數量控制在10個以下！感謝大家的提交！

衷心感謝所有參與訪談、問卷調查和文檔工作的朋友！你們的協作讓這一切成為可能。

### 你可以幫忙！

有許多方式可以協助我們寫出更完善的文件！

1. 若發現錯字、遇到指南問題或任何不盡完善之處，請點擊「編輯」按鈕並提交 PR。
2. [參與我們的問卷調查](https://www.surveymonkey.co.uk/r/DDZWQDJ)——這能幫助我們了解您如何使用 React Native 及其文件
3. 為我們撰寫內容！我們正在開發教學專區，以及離線應用程式、導航、無障礙功能、除錯、動畫、國際化與效能等主題指南。若您或認識的專家適合參與，請[與我聯繫](https://twitter.com/rachelnabors)！
