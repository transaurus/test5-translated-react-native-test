元件可以使用 Flexbox 演算法來指定其子元素的佈局。Flexbox 旨在為不同螢幕尺寸提供一致的佈局。

通常會結合使用 `flexDirection`、`alignItems` 和 `justifyContent` 來實現正確的佈局。

:::caution
Flexbox 在 React Native 中的運作方式與在網頁 CSS 中相同，但有一些例外。
預設值不同：`flexDirection` 預設為 `column` 而非 `row`，`alignContent` 預設為 `flex-start` 而非 `stretch`，`flexShrink` 預設為 `0` 而非 `1`，且 `flex` 參數僅支援單一數字。
:::

## Flex

[`flex`](layout-props#flex) will define how your items are going to **“fill”** over the available space along your main axis. Space will be divided according to each element's flex property.

在以下範例中，紅色、橙色和綠色視圖都是設有 `flex: 1` 的容器視圖的子元素。紅色視圖使用 `flex: 1`，橙色視圖使用 `flex: 2`，綠色視圖使用 `flex: 3`。**1+2+3 = 6**，這意味著紅色視圖將獲得 `1/6` 的空間，橙色視圖獲得 `2/6`，綠色視圖獲得 `3/6`。

```SnackPlayer name=Flex%20Example
import React from 'react';
import {StyleSheet, View} from 'react-native';

const Flex = () => {
  return (
    <View
      style={[
        styles.container,
        {
          // Try setting `flexDirection` to `"row"`.
          flexDirection: 'column',
        },
      ]}>
      <View style={{flex: 1, backgroundColor: 'red'}} />
      <View style={{flex: 2, backgroundColor: 'darkorange'}} />
      <View style={{flex: 3, backgroundColor: 'green'}} />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
  },
});

export default Flex;
```

## Flex Direction

[`flexDirection`](layout-props#flexdirection) 控制節點子元素的排列方向，也稱為主軸。交叉軸是與主軸垂直的軸，或是換行排列的軸。

- `column` (**預設值**) 從上到下排列子元素。如果啟用換行，則下一行將從頂部第一個項目的右側開始。
- `row` 從左到右排列子元素。如果啟用換行，則下一行將從左側第一個項目的下方開始。
- `column-reverse` 從下到上排列子元素。如果啟用換行，則下一行將從底部第一個項目的右側開始。
- `row-reverse` 從右到左排列子元素。如果啟用換行，則下一行將從右側第一個項目的下方開始。

你可以在此[了解更多](https://www.yogalayout.dev/docs/styling/flex-direction)。

## 佈局方向

佈局 [`direction`](layout-props#direction) 指定子元素和文字在層次結構中的排列方向。佈局方向也會影響 `start` 和 `end` 所指的邊緣。預設情況下，React Native 使用 LTR（從左到右）佈局方向。在此模式下，`start` 指的是左側，`end` 指的是右側。

- `LTR` (**預設值**) 文字和子元素從左到右排列。應用於元素起始的 margin 和 padding 會作用於左側。
- `RTL` 文字和子元素從右到左排列。應用於元素起始的 margin 和 padding 會作用於右側。

## Justify Content

[`justifyContent`](layout-props#justifycontent) 描述如何在容器的主軸上對齊子元素。例如，可以使用此屬性在 `flexDirection` 設為 `row` 的容器中水平居中子元素，或在 `flexDirection` 設為 `column` 的容器中垂直居中子元素。

- `flex-start`（**預設值**）將容器的子元素對齊至主軸的起始端。

- `flex-end` 將容器的子元素對齊至主軸的末端。

- `center` 將容器的子元素對齊至主軸的中心。

- `space-between` 沿主軸均勻分布子元素，剩餘空間分配在子元素之間。

- `space-around` 沿主軸均勻分布子元素，剩餘空間環繞子元素分配。與 `space-between` 相比，使用 `space-around` 會在第一個子元素前和最後一個子元素後也分配空間。

- `space-evenly` 沿主軸完全均勻分布子元素。相鄰子元素間距、主軸起始端與第一個子元素間距、主軸末端與最後一個子元素間距均完全相同。

了解更多請參閱[此處](https://www.yogalayout.dev/docs/styling/justify-content)。

## 對齊項目（Align Items）

[`alignItems`](layout-props#alignitems) describes how to align children along the cross axis of their container. It is very similar to `justifyContent` but instead of applying to the main axis, `alignItems` applies to the cross axis.

- `stretch`（**預設值**）拉伸子元素以匹配容器交叉軸的 `height`。

- `flex-start` 將子元素對齊至容器交叉軸的起始端。

- `flex-end` 將子元素對齊至容器交叉軸的末端。

- `center` 將子元素對齊至容器交叉軸的中心。

- `baseline` 沿共同基線對齊子元素。可為個別子元素設定基準線供父元素參考。

:::info
For `stretch` to have an effect, children must not have a fixed dimension along the secondary axis. In the following example, setting `alignItems: stretch` does nothing until the `width: 50` is removed from the children.
:::

了解更多請參閱[此處](https://www.yogalayout.dev/docs/styling/align-items-self)。

## 對齊內容（Align Content）

[alignContent](layout-props#aligncontent) 定義多行內容在交叉軸上的分布方式，僅在使用 `flexWrap` 換行時生效。

- `flex-start`（**預設值**）將換行內容對齊至交叉軸起始端。

- `flex-end` 將換行內容對齊至交叉軸末端。

- `stretch`（_網頁版 Yoga 的預設值_）拉伸換行內容以匹配容器交叉軸高度。

- `center` 將換行內容對齊至交叉軸中心。

- `space-between` 沿交叉軸均勻分布換行內容，剩餘空間分配在行間。

- `space-around` 沿交叉軸均勻分布換行內容，剩餘空間環繞各行分配。與 `space-between` 相比，使用 `space-around` 會在第一行前和最後一行後也分配空間。

了解更多請參閱[此處](https://www.yogalayout.dev/docs/styling/align-content)。

## Flex Basis、Grow 和 Shrink

- [`flexBasis`](layout-props#flexbasis) 是一種與軸無關的方式，用於提供項目在主軸上的預設大小。設置子元素的 `flexBasis` 類似於設置該子元素的 `width`（如果其父容器是 `flexDirection: row`）或 `height`（如果其父容器是 `flexDirection: column`）。項目的 `flexBasis` 是該項目的預設大小，即在任何 `flexGrow` 和 `flexShrink` 計算之前的大小。

- [`flexGrow`](layout-props#flexgrow) 描述容器內的主軸空間應如何在其子元素之間分配。在佈局其子元素後，容器會根據子元素指定的 flex grow 值分配任何剩餘空間。

  `flexGrow` 接受任何浮點數值 >= 0，預設值為 0。容器會根據子元素的 `flexGrow` 值加權分配剩餘空間。

- [`flexShrink`](layout-props#flexshrink) 描述當子元素的總大小在主軸上溢出容器大小時，如何沿主軸縮小子元素。`flexShrink` 與 `flexGrow` 非常相似，可以將其視為相同的方式，如果將任何溢出大小視為負剩餘空間。這兩個屬性也可以很好地協同工作，允許子元素根據需要增長和縮小。

  `flexShrink` 接受任何浮點數值 >= 0，預設值為 0（在網頁上，預設值為 1）。容器會根據子元素的 `flexShrink` 值加權縮小子元素。

了解更多[這裡](https://www.yogalayout.dev/docs/styling/flex-basis-grow-shrink)。

## 寬度和高度

The `width` property specifies the width of an element's content area. Similarly, the `height` property specifies the height of an element's content area.

Both `width` and `height` can take the following values:

- `auto`（**預設值**）React Native 會根據元素的內容（無論是其他子元素、文字還是圖片）計算元素的寬度/高度。

- `pixels` 以絕對像素定義寬度/高度。根據組件上設置的其他樣式，這可能是也可能不是節點的最終尺寸。

- `percentage` 以父元素的寬度或高度的百分比分別定義寬度或高度。

## 絕對與相對佈局

元素的 `position` 類型定義了它在其父元素中的定位方式。

- `relative`（**預設值**）預設情況下，元素採用相對定位。這意味著元素會根據正常的佈局流程進行定位，然後根據 `top`、`right`、`bottom` 和 `left` 的值相對於該位置進行偏移。此偏移不會影響任何兄弟元素或父元素的位置。

- `absolute` 當元素採用絕對定位時，它不會參與正常的佈局流程。相反，它會獨立於其兄弟元素進行佈局。其位置由 `top`、`right`、`bottom` 和 `left` 的值決定。
